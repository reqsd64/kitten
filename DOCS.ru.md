# Документация kitten

`kitten` печатает деревья файловой системы с метаданными элементов и
необязательным встроенным просмотром обычных текстовых файлов. Документ
описывает версию 0.1.0 в том виде, в котором она реализована в `src/main.c`,
`src/walk.c` и `src/output.c`.

## Требования

У программы нет сторонних зависимостей при запуске. Поддерживаемая основа:

- реализация C99;
- интерфейсы файловой системы и локалей POSIX.1-2008;
- POSIX-подобная операционная система.

Для сборки нужны компилятор C и `make`. Для необязательного руководства Info
также требуется программа `makeinfo` из Texinfo.

## Сборка

Обычная сборка создаёт `./kitten`:

```sh
make
```

Makefile учитывает стандартные переменные сборки:

```sh
make CC=clang CFLAGS='-O2 -pipe' LDFLAGS='-Wl,--as-needed'
```

По умолчанию `WARNINGS` содержит `-Wall -Wextra -Wpedantic`, а Makefile всегда
добавляет `-std=c99`. Прямая сборка того же набора исходных файлов выглядит
так:

```sh
cc -std=c99 -O2 -Wall -Wextra -Wpedantic \
  src/main.c src/walk.c src/output.c -o kitten
```

Удаление созданных файлов:

```sh
make clean
```

Цели Makefile:

| Цель | Результат |
|:--|:--|
| `all` | Собрать `kitten`; это цель по умолчанию. |
| `kitten` | Скомпилировать три исходника C после изменения исходника или `kitten.h`. |
| `info` | Создать `doc/kitten.info` из `doc/kitten.texi`. |
| `install` | Собрать и установить исполняемый файл и man-страницу. |
| `install-info` | Собрать и установить руководство Info. |
| `uninstall` | Удалить все три установленных файла. |
| `clean` | Удалить `kitten` и созданный `doc/kitten.info`. |

Для окружения сборки пакета можно переопределить `INSTALL`, `MAKEINFO` и `RM`.
Исполняемый файл компилируется одним запуском компилятора; промежуточных целей
объектных файлов нет.

## Установка

Префикс по умолчанию — `/usr/local`. `make install` устанавливает исполняемый
файл и man-страницу:

```sh
make
make install
```

Задайте другую иерархию через `PREFIX`, а каталог подготовки пакета — через
`DESTDIR`:

```sh
make PREFIX=/usr
make PREFIX=/usr DESTDIR="$pkgdir" install
```

Руководство Info необязательно:

```sh
make info
make PREFIX=/usr DESTDIR="$pkgdir" install-info
```

`make uninstall` удаляет исполняемый файл, man-страницу и руководство Info из
выбранных `PREFIX` и `DESTDIR`, но оставляет родительские каталоги.

## Командная строка

```text
kitten [ПАРАМЕТР]... [ПУТЬ]...
```

Без пути `kitten` просматривает текущий каталог. Параметры можно указывать до
или после путей. `--` завершает разбор параметров, поэтому он нужен перед
путём, начинающимся с `-`. Одиночный `-` считается путём.

Если передан ровно один каталог, `kitten` безопасно входит в него и печатает
корень как `./`. При нескольких операндах каждый путь выводится отдельным
корнем без смены рабочего каталога.

Параметры одного режима применяются слева направо. Поэтому из `--ascii` и
`--unicode` действует последний, а `--content=auto` может снова включить
просмотр после `--no-content`.

## Параметры

| Параметр | По умолчанию | Действие |
|:--|:--|:--|
| `-m БАЙТЫ`, `--max-bytes=БАЙТЫ` | `256K` | Максимальный размер просмотра одного обычного файла. |
| `-L ГЛУБИНА`, `--depth=ГЛУБИНА` | без ограничения | Ограничить спуск неотрицательной глубиной. |
| `--exclude=ШАБЛОН` | нет | Пропускать совпавшие элементы каталогов; можно повторять. |
| `--no-content` | выключен | Не показывать содержимое обычных файлов. |
| `--content=auto\|never` | `auto` | Включить обычную политику просмотра или отключить содержимое. |
| `--raw-content` | выключен | Не экранировать управляющие данные в тексте просмотра. |
| `--dirs-only` | выключен | Показывать и обходить только настоящие каталоги. |
| `-H`, `--human-readable` | выключен | Выводить размеры в КиБ, МиБ, ГиБ или ТиБ. |
| `-U`, `--unsorted` | выключен | Использовать порядок файловой системы и ограниченную память каталога. |
| `--ascii` | зависит от локали | Рисовать дерево символами ASCII. |
| `--unicode` | зависит от локали | Рисовать дерево символами Unicode. |
| `--summary` | выключен | Вывести общие счётчики после обхода. |
| `--language=auto\|en\|ru` | `auto` | Выбрать язык сообщений и подписей. |
| `--version` | — | Вывести версию и лицензию, затем завершиться. |
| `-h`, `--help` | — | Вывести справку, затем завершиться. |

Параметры со значением принимают отдельный операнд или длинную форму
`--имя=значение`. Короткие параметры нельзя объединять.

`БАЙТЫ` — беззнаковое десятичное число с необязательным суффиксом `K`, `M` или
`G`. Регистр суффикса не важен, множители являются степенями 1024. Значения,
не помещающиеся в `size_t`, отклоняются.

`ГЛУБИНА` — беззнаковое десятичное целое не больше `LONG_MAX`:

- `-L 0` печатает только корень;
- `-L 1` включает непосредственных потомков, но не входит в них;
- без параметра глубина не ограничена.

### Значения по умолчанию и правила разбора

Полное начальное состояние параметров:

| Настройка | Начальное значение |
|:--|:--|
| лимит просмотра | 262144 байта |
| максимальная глубина | без ограничения (внутреннее значение `-1`) |
| просмотр содержимого | включён |
| сырой вывод содержимого | выключен |
| только каталоги | выключен |
| удобочитаемые размеры | выключены |
| несортированный обход | выключен |
| исключения | пустой список |
| язык сообщений | определяется по локали |
| стиль дерева | Unicode только при кодировке локали UTF-8 |

Разборщик принимает только написания из таблицы параметров:

- короткие параметры нельзя объединять (`-HU` недопустим) или склеивать со
  значением (`-m64K` недопустим);
- длинные параметры со значением принимают `--имя значение` и
  `--имя=значение`;
- числа должны начинаться с цифры ASCII: знак, пробелы, дробная часть и
  посторонний текст недопустимы;
- размер принимает не более одного символа суффикса, поэтому `64KiB`, `1KB` и
  `1.5M` недопустимы;
- глубина не принимает суффиксы размера;
- пустой шаблон исключения недопустим;
- скалярные режимы следуют порядку командной строки, а каждый допустимый
  `--exclude` добавляет новый шаблон.

Неизвестные параметры и отсутствующие или недопустимые значения выводят одно
сообщение в стандартный поток ошибок и завершаются с кодом `1`. `--help` и
`--version` печатаются в стандартный вывод и завершают программу, как только
основной разбор доходит до них; последующие операнды не обрабатываются.

Диагностика разбора начинается с `kitten:` и использует одну из шести причин:
`недопустимый лимит просмотра`, `недопустимая глубина`, `недопустимый шаблон
исключения`, `недопустимый режим содержимого`, `недопустимый язык` или
`неизвестный параметр`. Если ошибочное значение доступно, оно экранируется и
добавляется к сообщению. Для отсутствующего значения аргумент не добавляется.

Операнды путей уплотняются в `argv` во время разбора. Поэтому параметры могут
следовать после путей, не меняя их порядок. После `--` все оставшиеся аргументы
считаются путями, даже если выглядят как параметры.

## Примеры

Просмотр текущего каталога:

```sh
kitten
```

Только дерево и метаданные:

```sh
kitten --no-content src
```

Не более двух уровней и файлы не больше 64 КиБ:

```sh
kitten -L 2 -m 64K path/to/project
```

Исключение обычного шума репозитория и итоговые счётчики:

```sh
kitten --exclude=.git --exclude='*.o' --exclude='build/*' --summary .
```

Воспроизводимые английские подписи и переносимые символы дерева:

```sh
kitten --language=en --ascii --no-content .
```

Несколько корней:

```sh
kitten README.md src LICENSE
```

Путь, начинающийся с дефиса:

```sh
kitten -- -example
```

## Дерево и метаданные

Сначала печатается каждый корень. Для потомков используются ветви Unicode
(`├──`, `└──`, `│`) или ASCII (`|--`, `` `-- ``, `|`).

Тип элемента определяется через `lstat`:

```text
имя/                                 каталог
имя [файл, 123 байта]                обычный файл
имя -> назначение [символическая ссылка]
имя [специальный файл, 0 байт]       любой другой тип
```

После каталогов ставится косая черта. Назначение символической ссылки
показывается, но переход по ссылке никогда не выполняется. К специальным
относятся FIFO, сокеты, устройства и другие элементы, не являющиеся каталогом,
обычным файлом или символической ссылкой.

Размер берётся из метаданных, полученных до просмотра. Без
`--human-readable` английский вывод использует `byte` или `bytes`, а русский —
подходящую форму слова «байт». В удобочитаемом режиме используются единицы IEC:
целое число для точных кратных и один десятичный знак в остальных случаях.

Несколько корней разделяются одной пустой строкой. У корня нет ветви дерева.
Потомок получает ветвь, а блок содержимого продолжается под её вертикалью:

```text
корень/
└── файл.txt [файл, 6 байт]
    ┌── содержимое
    │ привет
    └──
```

В режиме ASCII заголовок просмотра использует `+--`, строки содержимого — `|`,
а закрывающая строка — `` `-- ``.

### Потоки вывода

Обычные строки дерева, встроенные ошибки обхода, справка, сведения о версии и
итоги идут в стандартный вывод. Ошибки аргументов и невозможность войти в
единственный переданный каталог идут в стандартный поток ошибок.

Операционная ошибка, найденная после выбора элемента дерева, остаётся внутри
дерева, чтобы место сбоя было видно. Конечный код всё равно становится `1`.

### Встроенные отметки состояния

В русскоязычном дереве используются следующие отметки:

| Ситуация | Вывод |
|:--|:--|
| у каталога нет видимых потомков | `[пустой каталог]` |
| каталог нельзя открыть или прочитать | `[не удалось прочитать каталог: причина]` |
| каталог заменили перед открытием | `[каталог изменился во время открытия]` |
| метаданные пути нельзя прочитать | `[не найдено: причина]` |
| не удалось выделить память для элемента | `[недостаточно памяти]` |
| обычный файл пуст | `[пустой файл]` |
| обычный файл содержит нулевой байт | `[двоичный файл]` |
| текущий размер больше лимита | `[пропущено: размер РАЗМЕР превышает лимит ЛИМИТ]` |
| файл вырос сверх лимита при сканировании | `[пропущено: файл вырос сверх лимита ЛИМИТ]` |
| второй проход достиг лимита до EOF | `[обрезано после ЛИМИТ]` |
| обычный файл нельзя открыть | `[не удалось открыть: причина]` |
| обычный путь заменили перед открытием | `[файл изменился во время открытия]` |
| ошибка чтения или закрытия просмотра | `[ошибка чтения]` |

Если `readlink` завершается ошибкой, строка ссылки печатает системный текст
ошибки вместо назначения, всё равно добавляет `[символическая ссылка]`,
увеличивает счётчик ссылок и регистрирует ошибку.

## Обход

### Порядок

По умолчанию программа читает все видимые имена каталога, сортирует их через
`strcoll` в активной локали и затем обходит. Результат получается стабильным и
учитывает локаль, но имена каталога хранятся в памяти.

`--unsorted` сохраняет порядок файловой системы. На каждом открытом уровне
каталога остаётся только один ожидающий потомок, чтобы правильно нарисовать
последнюю ветвь. Это ограничивает память для элементов очень больших
каталогов; рекурсивные пути и буферы просмотра по-прежнему выделяются.

### Исключения

`--exclude` использует шаблоны POSIX `fnmatch`. Каждый шаблон сопоставляется с:

1. базовым именем элемента;
2. полным путём обхода, где `/` считается разделителем;
3. тем же путём без начального `./`, если он присутствует.

Например:

```sh
kitten --exclude=.git --exclude='*.o' --exclude='build/*' .
```

Исключённый каталог не печатается и не обходится. Шаблоны применяются к
элементам каталогов, но не к корневым операндам, явно указанным в командной
строке. Действуют правила `fnmatch`, а не `.gitignore`: `kitten` не добавляет
отрицание, поиск корня репозитория или неявную рекурсивную семантику `**`.

Каждый отфильтрованный элемент увеличивает итоговый счётчик `пропущено`.

### Режим каталогов

`--dirs-only` показывает только элементы, которые `lstat` определил как
настоящие каталоги. Обычные и специальные файлы, а также символические ссылки
— включая ссылки на каталоги — скрываются и считаются пропущенными.

Если при проверке видимости не удалось прочитать метаданные, элемент остаётся
видимым, чтобы обычный путь обхода сообщил об ошибке, а не молча отбросил его.

### Глубина

Корень имеет нулевую глубину. Ограничение управляет входом в каталог, а не его
отображением на предельном уровне. Поэтому `-L 1` печатает дочерние каталоги,
но не их содержимое.

### Символические ссылки

`kitten` использует `lstat` и никогда не входит в каталоги через символические
ссылки. Назначение читается в динамически растущий буфер, а не в буфер
фиксированного размера `PATH_MAX`. Ошибка чтения назначения выводится и
приводит к ненулевому коду завершения.

### Видимость и граничные случаи

- `.` и `..`, возвращённые `readdir`, игнорируются и не увеличивают
  `пропущено`.
- Каталог, все потомки которого исключены или скрыты `--dirs-only`,
  показывается пустым; скрытые элементы всё равно увеличивают `пропущено`.
- Достижение предельной глубины не считается пропуском. Каталог выводится без
  отметки пустоты, а его потомки не проверяются.
- Явно переданный корень не проверяется шаблонами исключений.
- Корень, не являющийся каталогом, при `--dirs-only` не создаёт строку дерева,
  увеличивает `пропущено` и завершается успешно, если нет другой ошибки.
- Специальные файлы описываются по метаданным `lstat` и никогда не открываются.
- Ошибка метаданных при проверке видимости для `--dirs-only` намеренно
  передаётся обычной обработке элемента, чтобы стать видимой ошибкой.

Сортированный обход заканчивает чтение и закрывает каталог до спуска в
собранных потомков. Если чтение завершилось ошибкой, уже собранные элементы
всё равно печатаются перед отметкой ошибки каталога. Несортированный обход
спускается, пока родительский поток открыт, а более позднюю ошибку `readdir`
или `closedir` сообщает после ожидающего потомка.

## Просмотр файлов

Содержимое показывается только для обычных файлов. Просмотр включён по
умолчанию и отключается через `--no-content` или `--content=never`.

Последовательность просмотра:

1. открыть путь, не переходя по конечной символической ссылке;
2. через `fstat` убедиться, что это всё ещё тот же обычный файл, который нашёл
   `lstat`;
3. сразу пропустить файл, если текущий размер в метаданных превышает лимит;
4. просканировать данные до лимита в поиске нулевых байтов и ошибок чтения;
5. вернуться к началу и напечатать файл построчно.

Файл с нулевым байтом во время сканирования помечается двоичным. Файл больше
лимита помечается пропущенным. Оба случая не считаются ошибками. Пустые файлы
получают отдельную подпись.

Изменения между первым сканированием и выводом могут быть замечены несколькими
способами. Замена файла, рост сверх лимита, ошибка чтения или слишком длинный
второй проход по возможности получают отдельный статус. Это всё равно
просмотр живой файловой системы, а не транзакционный снимок.

Перед строками содержимого печатаются направляющие дерева. Конечные байты CR и
LF удаляются, после чего `kitten` добавляет собственный перевод строки.
`--raw-content` отключает экранирование символов, но не превращает просмотр в
побайтовый `cat`: префиксы дерева и обработка строк сохраняются.

Максимальный единичный буфер просмотра ограничен заданным лимитом и длиной
выводимой строки. Весь файл не загружается в один непрерывный буфер.

### Таблица решений просмотра

Проверки упорядочены: раннее решение исключает последующую классификацию. В
частности, файл, уже превышающий лимит, не сканируется на нулевые байты.

| Обнаруженное условие | Результат просмотра | `пропущено` | `ошибки` |
|:--|:--|:--:|:--:|
| ошибка открытия | `не удалось открыть` | — | +1 |
| тип, устройство или inode открытого файла отличаются | `файл изменился во время открытия` | — | +1 |
| размер открытого файла превышает лимит | отметка лимита размера | +1 | — |
| сканирование нашло нулевой байт | `двоичный файл` | +1 | — |
| сканирование прочитало больше оставшегося лимита | отметка роста | +1 | — |
| ошибка сканирования или возврата к началу | `ошибка чтения` | — | +1 |
| сканирование достигло EOF на нуле байт | `пустой файл` | — | — |
| второй проход достиг лимита до EOF | отметка обрезания | +1 | — |
| ошибка чтения второго прохода или конечного закрытия | `ошибка чтения` | — | +1 |

Размер в строке метаданных файла приходит из начального `lstat`. Для решения о
лимите используется размер из `fstat` проверенного дескриптора. При изменении
файла на месте эти значения могут различаться.

Нулевой лимит просмотра допустим. Пустой файл сообщается как пустой; любые
данные, замеченные при сканировании или втором проходе, означают рост или
обрезание относительно нулевого лимита.

Сканирование читает блоками по 4096 байт и возвращается к началу через `fseek`.
При выводе буфер строки начинается с 256 байт или остатка лимита — меньшего из
двух значений — и растёт до этого лимита. Удаление CR и LF влияет только на
представление; эти байты входят в лимит.

## Безопасность вывода

Имена файловой системы могут содержать табуляцию, переводы строк, escape-байты,
некорректные многобайтовые последовательности и другие данные, меняющие вывод
терминала. Поэтому `kitten` всегда экранирует имена элементов и назначения
символических ссылок:

- `\n`, `\r` и `\t` используют именованные последовательности;
- обратная косая черта в именах и назначениях удваивается;
- управляющие символы ASCII и некорректные байты записываются как `\xNN`;
- многобайтовые управляющие символы — как `\uNNNN` или `\UNNNNNNNN`.

Обычный просмотр сохраняет табуляцию и печатаемые символы, распознанные
активной локалью. Управляющие символы ASCII, исходные байты диапазона C1,
некорректные кодировки и многобайтовые управляющие символы экранируются.
Обратные косые черты в тексте файла остаются без изменений.

`--raw-content` записывает текст просмотра без этих проверок. Доверенный файл
тогда может отправлять escape-последовательности терминалу. Параметр никогда не
отключает экранирование имён и назначений ссылок.

Диагностика может включать текст системной функции `strerror`; его формулировка
и кодировка зависят от библиотеки C и локали.

Обработка байтов по умолчанию:

| Вход | Имена и назначения ссылок | Текст просмотра |
|:--|:--|:--|
| табуляция | `\t` | исходная табуляция |
| LF или CR | `\n` или `\r` | удаляется как конец строки, иначе экранируется |
| обратная косая черта | удваивается | не меняется |
| байт ASCII меньше 32 или байт 127 | `\xNN` | `\xNN`, кроме табуляции просмотра |
| отдельный байт `0x80`–`0x9f` | `\xNN` | `\xNN` |
| некорректная или неполная многобайтовая последовательность | ошибочный байт как `\xNN` | так же |
| допустимый многобайтовый управляющий символ | `\uNNNN` или `\UNNNNNNNN` | так же |
| допустимый печатаемый символ | копируется | копируется |

`mbrtowc` и `iswcntrl` используют активную локаль. Состояние преобразования
сбрасывается после некорректной или неполной последовательности и после
управляющих символов ASCII, обработанных без `mbrtowc`; допустимые
преобразования сохраняют состояние.

## Гонки файловой системы

Обход намеренно отделяет метаданные для представления от безопасного открытия:

- `lstat` определяет элемент без перехода по символической ссылке;
- обычные файлы и каталоги открываются с `O_NONBLOCK` и `O_NOFOLLOW`;
- полученный дескриптор проверяется через `fstat`;
- тип, устройство и inode должны совпасть с предыдущим наблюдением.

Неблокирующее открытие не позволяет зависнуть, если обычный путь заменили FIFO
до проверки его типа. Дескриптор подтверждённого обычного файла возвращается в
блокирующий режим до начала чтения через stdio.

Та же сверка устройства и inode защищает удобный режим одного каталога перед
`fchdir`. Замеченная замена выводится как изменение. Эти проверки уменьшают
обычные риски между проверкой и использованием, но файл всё ещё может
измениться на месте уже после открытия.

## Итоги

`--summary` печатает общие значения после всех запрошенных корней:

```text
итого:
  каталоги: 3
  файлы: 7
  ссылки: 1
  специальные: 0
  байты: 68421
  пропущено: 2
  ошибки: 0
```

Значение счётчиков:

| Счётчик | Значение |
|:--|:--|
| `каталоги` | Показанные настоящие каталоги, включая корневые. |
| `файлы` | Показанные обычные файлы. |
| `ссылки` | Показанные символические ссылки. |
| `специальные` | Показанные элементы всех остальных типов. |
| `байты` | Сумма размеров обычных файлов из метаданных независимо от показа содержимого. |
| `пропущено` | Отфильтрованные элементы и просмотры, пропущенные как двоичные, слишком большие, выросшие или обрезанные. |
| `ошибки` | Ошибки метаданных, открытия, чтения, выделения памяти и обхода. |

Сумма байтов насыщается на `UINTMAX_MAX`; при достижении предела итог помечает
переполнение.

Поскольку `пропущено` объединяет отфильтрованные элементы и решения о
содержимом, это не только количество файлов.

Счётчики обновляются по показанному элементу, а не по успешности чтения
содержимого:

| Событие | Изменение счётчиков |
|:--|:--|
| показан настоящий каталог | `каталоги +1` |
| показан обычный файл | `файлы +1`, его начальный размер из метаданных добавляется к `байты` |
| показана символическая ссылка | `ссылки +1`, даже при ошибке чтения назначения |
| показан специальный файл | `специальные +1`; его размер не добавляется к `байты` |
| элемент исключён или скрыт `--dirs-only` | `пропущено +1` |
| просмотр двоичный, слишком большой, выросший или обрезанный | `пропущено +1` |
| предел глубины запретил спуск | нет изменений сверх показанного каталога |
| пустой файл или видимый пустой каталог | нет пропуска и ошибки |
| операция завершилась ошибкой | `ошибки +1` в месте сообщения |

Обычный файл остаётся учтённым, а его размер остаётся в `байты`, если
содержимое отключено, пропущено или не открылось. Путь с ошибкой `lstat` не
получает счётчик типа, потому что тип не был определён.

## Код завершения

| Код | Значение |
|:--:|:--|
| `0` | Параметры допустимы, каждый обход завершился без замеченных ошибок. |
| `1` | Возникли ошибки параметров, путей, обхода, просмотра или потоков вывода. |

Двоичные и слишком большие файлы пропускаются, но не считаются ошибками.
Ошибка любого корня делает конечный код равным `1`, при этом остальные части
по возможности продолжают обходиться. Ошибка сброса стандартного вывода или
стандартного потока ошибок также меняет код на `1`.

## Локализация и символы дерева

При запуске `setlocale(LC_ALL, "")` активирует окружение процесса. В
автоматическом режиме локаль `ru` выбирает русские подписи и сообщения, а
остальные локали — английские. Стандартный приоритет (`LC_ALL`,
`LC_MESSAGES`, затем `LANG`) обрабатывается библиотекой C.

`--language=en` и `--language=ru` задают воспроизводимые сообщения.
`--language=auto` возвращает определение по локали. Имена и допустимые значения
параметров всегда остаются английскими и не переводятся.

Стиль дерева выбирается отдельно от языка сообщений. Локаль, для которой
кодировка сообщается как UTF-8 или UTF8, использует ветви Unicode; остальные
кодировки — ASCII. `--ascii` и `--unicode` переопределяют выбор.

Перед обычным разбором аргументов `main.c` ищет допустимый параметр языка. Это
позволяет параметру в конце строки управлять диагностикой более раннего
недопустимого параметра.

Определение русской локали намеренно узкое: имя должно начинаться с `ru` без
учёта регистра, после чего допустим конец строки, `_`, `-` или `.`. Для дерева
кодировка без учёта регистра сравнивается с `UTF-8` или `UTF8`. Если активация
локали не удалась, продолжает действовать текущая локаль библиотеки C.

Предварительный поиск языка останавливается на `--` и пропускает следующий
аргумент каждого известного параметра со значением. Поэтому путь или значение,
равное `--language`, не принимается за параметр языка. Недопустимое значение
языка не меняет предварительный выбор; основной проход позднее сообщает об
ошибке.

Локализованные подписи предоставляет программа. Причины ошибок из `strerror`
приходят от системы и могут оказаться на другом языке, если у неё нет
подходящего каталога сообщений.

### Общее состояние

`struct kitten_options` содержит полностью разобранные режимы и принадлежащие
ему шаблоны исключений. `struct kitten_totals` накапливает счётчики и признак
переполнения суммы байтов. `struct kitten_context` передаёт указатели на обе
структуры по рекурсивному обходу. Изменяемого глобального состояния нет.

Поля `kitten_options` напрямую соответствуют состоянию командной строки:

| Поле | Значение |
|:--|:--|
| `preview_limit` | Лимит байтов одного файла. |
| `max_depth` | Неотрицательный предел или `-1` без ограничения. |
| `show_content` | Запускают ли обычные файлы просмотр. |
| `raw_content` | Отключено ли экранирование просмотра. |
| `dirs_only` | Фильтруются ли элементы, не являющиеся каталогами. |
| `unicode_tree` | Символы дерева Unicode вместо ASCII. |
| `human_readable` | Единицы IEC вместо точного количества байтов. |
| `unsorted` | Порядок файловой системы вместо сбора и сортировки. |
| `russian` | Русские строки программы вместо английских. |
| `exclude_patterns` | Принадлежащий структуре массив копий шаблонов `fnmatch`. |
| `exclude_count` | Количество элементов этого массива. |

`kitten_totals` содержит `directories`, `files`, `symlinks`, `special`,
`skipped`, `errors` и `bytes` типа `uintmax_t`, а также флаг
`bytes_overflow`. `kitten_context` только связывает постоянные параметры с
изменяемыми итогами.

Автоматы просмотра используют три внутренних перечисления: `preview_open`
различает успех, системную ошибку открытия и изменение идентичности;
`preview_scan` — текст, двоичные данные, рост и ошибку чтения; `preview_line` —
EOF, полную строку, обрезание и ошибку.

### Поток выполнения

1. `main` инициализирует значения по умолчанию и локаль.
2. Предварительный проход выбирает язык диагностики аргументов.
3. Основной разбор обновляет параметры и уплотняет операнды путей в `argv`.
4. Единственный каталог проверяется, открывается и становится текущим; все
   остальные корни напрямую передаются в `kitten_walk_path`.
5. `print_entry` классифицирует каждый путь через `lstat`.
6. Каталоги выбирают сортированный или несортированный обход; обычные файлы
   могут запустить просмотр; ссылки и специальные файлы завершаются выводом
   метаданных.
7. Ошибки увеличивают счётчики и передаются через возвращаемые значения, не
   прерывая несвязанные корни.
8. `main` при необходимости выводит итоги, сбрасывает оба потока, освобождает
   шаблоны и возвращает код 0 или 1.

### Алгоритмы каталогов

Сортированный обход хранит копии всех видимых имён, сортирует массив указателей
и затем создаёт пути потомков. Память уровня каталога линейна по количеству и
общей длине видимых имён.

Несортированный обход создаёт пути по мере чтения и хранит один ожидающий путь,
чтобы определить последний элемент. Память ожидающего элемента на уровне
каталога постоянна, если не учитывать длину пути. Он может держать один поток
каталога на каждом рекурсивном уровне. Сортированный обход закрывает поток до
спуска. Оба алгоритма используют динамические префиксы и пути вместо буферов
фиксированного размера.

### Модель ошибок

Функции возвращают ненулевое логическое значение при ошибке и одновременно
увеличивают `kitten_totals.errors` в точке, где ошибка стала видимой. Благодаря
этому обход печатает максимум полезного контекста, сохраняя неуспешный код
процесса.

Нехватка памяти обрабатывается локально. Если возможно, соответствующая строка
дерева получает отметку о нехватке памяти, а соседние элементы продолжают
обрабатываться. Ошибка создания состояния, необходимого для чтения каталога,
завершает эту часть обхода.

### Владение ресурсами и инварианты

- `main` владеет строками исключений от успешного разбора до
  `finish_program`.
- Сортированный обход владеет копиями имён, пока не была предпринята обработка
  каждого собранного потомка, в том числе после ошибки `readdir`.
- Каждый объединённый путь и префикс дерева освобождает создавший его кадр
  стека.
- `fdopendir` принимает владение проверенным дескриптором каталога; `closedir`
  освобождает его.
- `fdopen` принимает владение проверенным дескриптором обычного файла;
  `fclose` освобождает его.
- Дескриптор в `enter_directory` закрывается после `fchdir`; новый рабочий
  каталог продолжает действовать независимо от дескриптора.
- Буфер строки просмотра и буфер назначения ссылки освобождаются на каждом
  завершённом пути их владельцев.

Успешные функции безопасного открытия сохраняют центральный инвариант:
дескриптор имеет ожидаемый тип и ту же пару `(st_dev, st_ino)`, которую ранее
вернул `lstat`. Несовпадение сообщается как изменение, а не как общая системная
ошибка.

Вокруг вызовов `readdir` значение `errno` сбрасывается, чтобы отличить конец
каталога от ошибки. Ошибка закрытия сохраняется, если тот же поток ещё не имеет
предшествующей ошибки чтения.

## Переносимость и ограничения

- Нативные API Windows не поддерживаются. Требуется POSIX-совместимое
  окружение.
- Сортировка и определение печатаемых символов зависят от активной локали.
- Определение двоичных данных намеренно простое: нулевой байт означает двоичный
  файл. MIME-анализ и перекодирование отсутствуют.
- Исключения являются шаблонами POSIX, а не правилами систем контроля версий.
- У рекурсии каталогов нет отдельного ограничения глубины стека; для
  недоверенных или очень глубоких деревьев используйте `-L`.
- При одновременных изменениях метаданные и содержимое могут относиться к
  разным моментам.
- Обычные файлы с жёсткими ссылками считаются и просматриваются по каждому
  видимому пути.
- Вывод предназначен для людей и текстовых снимков; схемы JSON или другого
  стабильного машинного формата нет.
- Используются только коды завершения 0 и 1.

Реализация избегает фиксированных буферов `PATH_MAX` и GNU-специфичного разбора
командной строки. В переносимую основу входят `readdir`, `fdopendir`, `fnmatch`,
`lstat`, `readlink`, `fchdir` и неблокирующий `open`.

## Подготовка пакетов

Дистрибутивы должны передавать свою политику компиляции через `CC`, `CPPFLAGS`,
`CFLAGS`, `LDFLAGS` и `LDLIBS`, а установку выполнять в подготовительный
каталог:

```sh
make PREFIX=/usr
make PREFIX=/usr DESTDIR="$pkgdir" install
```

Пакету для запуска нужны только исполняемый файл и man-страница. Руководство
Info можно собрать и упаковать отдельно при наличии Texinfo:

```sh
make info
make PREFIX=/usr DESTDIR="$pkgdir" install-info
```

Используйте идентификатор SPDX `BSD-2-Clause`. Рекомендации целевого
дистрибутива или коллекции не должны менять классификацию лицензии проекта.
